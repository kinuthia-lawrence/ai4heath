# Model Validation Metrics Explained

This document explains the validation metrics generated by the model training pipeline in `train.py`. These metrics are saved in `validation_metrics.csv` after each training run.

## Metrics Overview

The following metrics are reported for each class (disease) and as overall averages:

- **Precision**: The proportion of positive identifications that were actually correct. High precision means that when the model predicts a certain disease, it is usually right.
  - Formula: $\text{Precision} = \frac{\text{True Positives}}{\text{True Positives} + \text{False Positives}}$

- **Recall**: The proportion of actual positives that were correctly identified. High recall means the model finds most cases of a disease.
  - Formula: $\text{Recall} = \frac{\text{True Positives}}{\text{True Positives} + \text{False Negatives}}$

- **F1-score**: The harmonic mean of precision and recall. It balances the two, especially useful when classes are imbalanced.
  - Formula: $\text{F1} = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$

- **Support**: The number of true instances for each class in the validation set. This shows how many samples were used to compute the metrics for each class.

- **Accuracy**: The overall proportion of correct predictions (across all classes).
  - Formula: $\text{Accuracy} = \frac{\text{Number of Correct Predictions}}{\text{Total Predictions}}$

- **Macro avg**: The average of the metric (precision, recall, F1) across all classes, treating each class equally.
- **Weighted avg**: The average of the metric across all classes, weighted by the number of samples (support) in each class.

## How to Use

- After running `train.py`, open `validation_metrics.csv` to review the model's performance.
- Use these metrics to compare different models or training runs.
- High values (close to 1.0) indicate excellent performance, but always check support to ensure enough samples per class.
